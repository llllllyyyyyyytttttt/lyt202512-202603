# 一、问题本质：拒绝推断到底在解决什么？

## 1️⃣ 信贷真实世界结构

在实际申请人中有三类人：

* 批准 + 还款好
* 批准 + 坏账
* 拒绝 + （未知好坏）

我们能观察到：

✔ 批准人群的真实违约
✖ 拒绝人群的真实违约

因此：

> 训练样本 ≠ 全量申请人

这叫 **选择偏差（Selection Bias）**

---

# 二、拒绝推断的真正目标

很多人理解错。

❌ 不是为了让拒绝样本有标签
❌ 不是为了单纯提升 KS

真正目标是：

> 让模型学到更接近“全量申请人风险结构”的排序能力

注意两个关键词：

* 全量
* 排序结构

---

# 三、为什么直接训练是有偏的？

因为模型看到的是：

> 被风控筛过的人群

这会导致：

* 极高风险人被过滤
* 极低风险人被过滤
* 中间风险密集

模型学到的是“被筛选后的风险结构”。

这就是：

**样本选择偏差**

---

# 四、拒绝推断方法体系

## 方法一：简单外推（最常见）

用现有模型预测拒绝样本：

```
p_reject = model(x_reject)
```

然后：

* 按比例分配好坏
* 或直接当软标签

优点：

* 简单
* 快

缺点：

* 强依赖原模型
* 容易自我强化偏差

---

## 方法二：分层比例法（Parceling）

步骤：

1. 按评分分箱
2. 计算每箱批准人坏账率
3. 将同箱拒绝样本按比例赋值

核心假设：

> 同分数段，批准与拒绝违约率相同

风险：

这个假设往往不成立。

---

## 方法三：重加权法（Reweighting）

思想：

估计一个人被批准的概率：

```
P(S=1 | X)
```

然后加权训练：

```
weight = 1 / P(S=1|X)
```

这是理论上更严谨的方法。

但：

实现复杂
稳定性要求高

---

# 五、拒绝推断的风险点（非常重要）

## 1️⃣ 噪音标签放大

拒绝样本没有真实标签
一旦赋错，会：

* 污染训练集
* 降低泛化能力

---

## 2️⃣ 结构性漂移

拒绝样本特征分布往往不同。

加入训练后：

* PSI 变高
* 排序结构扭曲

---

## 3️⃣ 截面提升但非截面变差

原因：

拒绝样本标签是估计的
时间维度会放大误差

---

# 六、如何评估拒绝推断是否成功？

你必须同时满足 5 条：

---

## ① 截面 KS 提升

说明当期排序变好。

---

## ② 非截面 KS 不下降

说明时间稳定性没被破坏。

---

## ③ PSI 不显著上升

说明分布结构没被扭曲。

---

## ④ Lift 前端更集中

说明真正业务价值提升。

---

## ⑤ 拒绝样本风险排序合理

例如：

* 高分拒绝样本违约率估计合理
* 分层违约率单调

# 七、完整实战流程（推荐你按这个做）

---

## Step 1：做 baseline

只用批准样本训练。

记录：

* KS
* Lift
* PSI
* 截面 / 非截面

---

## Step 2：构建拒绝推断版本

使用一种方法：

* 外推
* 分层比例
* 或重加权

---

## Step 3：合并训练

重新训练模型。

---

## Step 4：严格对比

必须对比：

| 维度       | baseline | RI模型 |
| -------- | -------- | ---- |
| 截面KS     |          |      |
| 非截面KS    |          |      |
| PSI      |          |      |
| Lift前20% |          |      |

---

## Step 5：判断

只有同时满足：

* KS 提升
* 非截面稳定
* PSI 可控

才可以进入上线评估。

否则：

回滚。

---

# 八、什么时候拒绝推断值得做？

以下情况强烈建议做：

* 拒绝率 > 50%
* 批准样本偏窄
* 市场策略变动大

不建议做：

* 批准率本身很高
* 样本量巨大
* 时间稳定性优先

---

# 九、拒绝推断的高级理解

它本质上是在解决：

> Missing Not At Random (MNAR)

拒绝不是随机缺失。

它是：

被模型规则选择性过滤的缺失。

所以理论上：

不存在完美解决方案。

只有“权衡”。

---

# 十、给你一个最终心法总结

拒绝推断不是：

提升KS的技巧。

而是：

在“结构偏差”与“标签噪音”之间找平衡。

你必须：

像风控负责人一样看问题
而不是像建模工程师一样只看指标。
